* VarMap Missing Identifier Types Analysis

** Current Implementation Status

The VarMap tool currently handles the following identifier contexts correctly:
- Variable declarations (var, let, const)
- Function declarations and named function expressions
- Function parameters (in formal parameter lists)
- Class declarations
- Basic identifier references

** Missing Identifier Contexts

After comprehensive testing, the following JavaScript identifier contexts are NOT being canonicalized:

*** 1. Destructuring Patterns
**** Object destructuring
#+BEGIN_SRC javascript
const { prop1, prop2 } = obj;  // prop1, prop2 not canonicalized
const { prop: renamed } = obj;  // renamed not canonicalized
const { nested: { deep } } = obj;  // deep not canonicalized
#+END_SRC

**** Array destructuring
#+BEGIN_SRC javascript
const [elem1, elem2] = arr;  // elem1, elem2 not canonicalized
const [, , third] = arr;  // third not canonicalized
const [first, ...rest] = arr;  // rest not canonicalized
#+END_SRC

**** Function parameter destructuring
#+BEGIN_SRC javascript
function fn({ param1 }, [param2]) {}  // param1, param2 not canonicalized
const fn = ({ param1 }) => {};  // param1 not canonicalized
#+END_SRC

*** 2. Loop Variables
**** For-in loops
#+BEGIN_SRC javascript
for (const key in obj) {}  // key not canonicalized
#+END_SRC

**** For-of loops
#+BEGIN_SRC javascript
for (const value of array) {}  // value not canonicalized
#+END_SRC

*** 3. Catch Clause Parameters
#+BEGIN_SRC javascript
try {} catch (error) {}  // error not canonicalized
#+END_SRC

*** 4. Arrow Function Parameters
#+BEGIN_SRC javascript
const fn = (param1, param2) => {};  // param1, param2 not canonicalized
#+END_SRC

*** 5. Function Expression Parameters
#+BEGIN_SRC javascript
const fn = function(param) {};  // param not canonicalized
#+END_SRC

*** 6. Import Bindings
#+BEGIN_SRC javascript
import { imported } from './mod';  // imported not canonicalized
import * as ns from './mod';  // ns not canonicalized
import defaultImport from './mod';  // defaultImport not canonicalized
#+END_SRC

*** 7. Method Definitions in Objects
#+BEGIN_SRC javascript
const obj = {
  method(param) {}  // method scope and param not handled properly
};
#+END_SRC

** Correctly Handled Non-Variable Contexts

The following are correctly NOT canonicalized (as they shouldn't be):
- Object property names in dot notation (obj.property)
- Object property names in declarations ({ property: value })
- String literals in bracket notation (obj['property'])
- Export aliases (export { var as alias })
- Labels (labelName: for ...)
- Property names in shorthand ({ prop }) - the property name itself

** Implementation Notes

The main issues are in the scope analyzer (/home/shcv/projects/varmap/src/scope/mod.rs):
1. It only checks for specific node types in the visit_node method
2. It doesn't handle destructuring patterns at all
3. It doesn't handle loop variable declarations
4. It doesn't handle catch clauses
5. Arrow function and function expression parameters are not properly extracted

The canonicalizer seems to work correctly for identifiers that are properly detected by the scope analyzer, so the primary fix needed is in the scope analysis phase.

** Recommendations

1. Add destructuring pattern support to handle_variable_declaration
2. Add new handlers for:
   - for_in_statement
   - for_of_statement  
   - catch_clause
   - import_statement
   - object method definitions
3. Fix arrow function parameter extraction
4. Fix function expression parameter extraction
